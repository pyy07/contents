OpenAI 颠覆式实验：5个月零手写代码，AI承包了百万行软件工程

当工程师不再写代码，他们还能做什么？OpenAI 用 5 个月的实验给出了答案。

一个疯狂的想法

2025年8月，OpenAI 团队做了一个大胆的实验：

构建一款软件产品，一行代码都不由人类手写。

5个月后，结果令人震惊：

  • 📦 约100万行代码
  • 👥 3名工程师 → 平均每天3.5个PR
  • ⏱️ 只用1/10的传统开发时间
  • 🏠 已有数百名内测用户

这不是概念验证，而是一个真正上线的产品——有用户、有部署、有故障、有修复。

唯一的不同是：每一行代码，都是由AI写的。

工程师的新角色

当你不再写代码，工程师的工作变成了什么？

OpenAI 总结了一个词：Harness（驾驭）

核心转变：从"写代码的人"变成"构建环境的人"

具体来说：

  1. 拆分任务——把大目标拆成小模块（设计、代码、评审、测试）
  2. 编写提示——告诉AI怎么构建这些模块
  3. 构建反馈循环——当AI失败时，问"缺了什么能力？如何让它可理解？"
这里的关键词是环境。OpenAI发现：瓶颈从来不是AI写代码的能力，而是缺乏结构、工具和反馈机制。

让AI"看见"系统

传统开发中，人类通过日志、监控来理解系统。AI也需要。

OpenAI做了几件事：

1. 让AI能启动应用

  • 每个Git worktree都能独立运行
  • AI可以为每个改动启动一个独立实例测试

2. 集成浏览器能力

  • 接入Chrome DevTools Protocol
  • AI可以直接看DOM快照、截图
  • 能够复现Bug、验证修复、推理UI行为

3. 可观测性栈

  • 日志用LogQL查
  • 指标用PromQL查
  • AI可以主动检查"服务启动是否在800ms内"
结果：单个AI任务可以连续工作6个小时（通常在人类睡觉时）。

知识管理：不要1000页手册

OpenAI踩过的坑：

  • ❌ 一个巨大的AGENTS.md → 上下文被稀释，AI不知道什么重要
  • ❌ 放在Google Docs的文档 → AI根本看不到
  • ❌ 放在人类头脑中的知识 → 对AI完全不可见

正确的做法：

把代码仓库变成唯一的"事实来源"（System of Record）
  • AGENTS.md只做"目录"（约100行）
  • 真正的知识放在结构化的docs/目录
  • 用linters自动检查文档是否过时
  • 定期有AI扫描并修复过时文档

架构约束：AI时代的必备条件

你可能以为"严格架构"是大公司才需要的。

OpenAI说：AI时代，这是早期必要条件。

他们的规则：

  • 每个业务域分固定层级：Types → Config → Repo → Service → Runtime → UI
  • 跨域依赖只能通过单一接口（Providers）
  • 用自定义linter机械执行
"这通常是你有几百个工程师后才做的。AI时代，它是先决条件。"

为什么？因为没有约束，AI会跑偏。约束不是限制AI，而是让AI跑得更快。

吞吐量改变一切

当AI可以每天生成几十个PR时，很多传统工程规范变得反效率：

  • ❌ 长时间Review → AI可以自己Review自己
  • ❌ 阻塞式测试 → 重跑成本低，等待成本高
  • ❌ 大改动 → 小改动更容易成功
在一个AI吞吐量远超人类注意力的系统里，等待是最贵的。

熵与垃圾回收

完全自主的AI也带来新问题：AI会复制已有的坏习惯。

OpenAI把它叫作"AI垃圾"——技术债务。

他们的解法：

  1. 编码"黄金原则"——带主观意见的机械规则
  2. 后台AI扫描——定期检查代码质量，发起重构PR
  3. 像垃圾回收一样——小额持续偿还债务
"技术债务就像高息贷款。持续小额还款，比一次性痛苦还清要好得多。"

什么是真正的"AI原生"开发？

当OpenAI说"AI写的代码"，他们指的是整个代码库：

  • 产品代码 + 测试
  • CI配置 + 发布工具
  • 内部开发者工具
  • 文档 + 设计历史
  • 评估框架
  • 审查评论 + 回复
  • 管理脚本

最近，AI甚至能端到端驱动新功能：

  1. 验证代码库状态
  2. 重现Bug
  3. 录制故障视频
  4. 实施修复
  5. 验证修复
  6. 打开PR
  7. 回应反馈
  8. 合并更改

全程无需人类介入。

启示

OpenAI的实验揭示了一个趋势：

未来的软件工程师，核心技能不再是写代码，而是：设计环境、定义意图、构建反馈循环。

软件工程依然需要纪律——但这种纪律更多体现在支撑结构上，而不是代码本身。

工具、抽象、反馈回路——这些变得前所未有的重要。
